Minimum Spanning Trees

Given a undirected graph G with positive edge weights

Def: A spanning tree of G is a subgraph T that is both a tree (connected and acyclic) and spanning (including all of the vertices of G)

Goal: Find a minimum weight spanning tree (MST) of G

Greedy algorithm

Simply assumptions: Edge weights are distinct

Graph is connected

Consequence: MST is exisit and unique

Cut: A partition of the vertices of G into two nonempty sets

Crossing edge: An edge that connects a vertex in one set of a cut to a vertex in the other set

Cut property: Given any cut, the crossing edge of minimum weight is in the MST

Proof:

  • Suppose min-weight crossing edge e is not in the MST

  • Adding e to MST creates a cycle.

  • Some other edge f in the cycle must be a crossing edge

  • Removing f from the MST and adding e yields a spanning tree with smaller weight

Greedy MST algorithm demo

  • Start with all edges colored gray

  • Find cut with no black edges; color min-weight crossing edge black

  • Repeat until V-1 edges are colored black

Proof:

  • Any edge colored black is in the MST

  • Fewer than V-1 edges black edges => cut with no black edges

  • (consider cut whose vertices are on connected comopnent)

the problem is what to choose to cut and how to find the min-weight edge

two assumptions

if the edge weight are not distinct the greedy algorithm still work

if the graph is not connected, the greedy algorithm compute each connected component’s MST

Edge-weighted graph API

Kruskal’s algorithm

Consider edges in ascending order of weight

Add edge to tree T unless doing so would create a cycle

Kruskal algorithm is a speacil case of greedy algorithm

  • Suppose kurusakal algorithm colors the edge e (v -> w) black

  • Cut = set of vertices connected to v in tree T

  • No crossing edge is black

  • No crossing edge has lower weight

Challenge: How to check if adding e to T creates a cycle?

  • union-find data structure

  • maintain a data structure to keep track of connected components

  • if a new edge connects two vertices in the same component, mean it will create a cycle

in time proportional to E log E

Prim’s algorithm

  • Start with vertex 0 and greedily grow tree T

  • Add to T the min-weight edge with exactly one endpoint in T

  • Repeat until V-1 edges

challenge: how to find the min-weight edge with exactly one endpoint in T

Lazy version :

  • maintain a priority queue of edges with at least one endpoint in T

  • Delete min edge v-w from priority queue

  • Disregard if both endpoints v and w are in T

  • Otherwise, let w be the vertex not in T

  • Add to PQ any edge incident to w

  • Add w to T

in time proportional to E log E

Eager version :

  • maintain a priority queue of vertices conntected by an edge in T where the priority of v is the weight of the min-weight edge connecting v to T

  • Delete min vertex v from priority queue add and its associated edge e = v-w from T

  • Update PQ by considering all edges e = v-x incident to v

  • ignore if x is already in T

  • add x to PQ if not already on PQ

  • decrease priority of x if v->x becomes shortest edge connecting x to T

use keys[i] qp[i] and pq[i] to maintain the priority queue
