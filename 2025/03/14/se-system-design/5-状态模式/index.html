状态模式

状态模式(State Pattern)

允许一个对象在其内部状态改变的时候改变其行为，对象看来像是修改了它的类。

State Pattern: Allow an object to alter its behavior when its internal state changes. The object will appear to change its class.

当状态之间没有关联（简单状态模式）的时候实际上就是策略模式

模式结构

classDiagram
    class Context {
        -State s
        +setState(State s)
        +run()
    }
    class State {
        +run()
    }
    class ConcreteStateA {
        +run()
    }
    class ConcreteStateB {
        +run()
    }

    Context o--> State
    State <|-- ConcreteStateA
    State <|-- ConcreteStateB

包含如下角色

Context: 环境类： 实际上就是拥有状态的对象，环境类可以充当状态管理器的角色，可以在环境类中对状态进行切换操作

State：抽象状态类

ConcreteState: 具体状态类

与策略模式很相似，但是仅仅是类图相似，策略模式中一个Context实际上只对象一个具体的Strategy，但是在状态模式中Context是和多个State动态对应的

优缺点

优点：

封装了转换规则

枚举可能的状态

可以方便地增加新地状态

状态转换逻辑和状态对象合成一体

让多个环境对象共享一个状态对象，减少系统中对象的个数

缺点：

增加系统类和对象的个数

使用不当将导致程序结构和代码的混乱

对开闭原则的支持不太好

适用环境

对象行为依赖其状态/根据其状态改变而改变其相关行为

包含大量与状态有关的条件语句

模式扩展

  1. 简单状态模式：状态都相互独立，状态之间无需进行转换的状态模式，其遵循开闭原则

  2. 可切换状态的状态模式：在具体的状态类内部需要调用环境类Context setState() 进行状态转换。 新增的状态类可能需要修改其他某些状态类甚至环境类的源代码
