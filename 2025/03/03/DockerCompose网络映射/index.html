DockerCompose网络映射

docker容器自身有一个export的端口，负责与外部通信, 容器内部的应用监听这个端口。例如EXPOSE 8080

docker-compose可以用来自动进行容器编排。其原理是创建一个虚拟网络，为不同的docker容器分配不同的IP地址。同时docker-compose提供DNS服务，将容器的名字映射到容器的IP地址上。容器之间可以直接通过容器名字访问。

例如简单创建一个nginx容器，监听80端口。

使用docker-compose.yml文件进行配置

使用docker-compose up -d启动服务

docker-compose在启动这个服务的同时将本地8081端口映射到docker容器ip, 因此宿主机可以通过http://localhost:8081访问nginx容器。

docker-compose首先创建了一个虚拟网络。

使用ip a查看虚拟网络，可以看到有一个br-开头的虚拟网络

使用sudo iptables -t nat -L -n -v查看nat表，可以看到docker-compose创建了一个名为DOCKER的链，pre-routing和output将target设置为这个DOCKER链。DOCKER链指明了所有来源不是本地的数据包都会被重定向即172.23.0.2:80(我的机器)

这就是为什么可以通过localhost:8081访问nginx容器的原因。事实上直接访问172.23.0.2:80也是可以的。

可以通过docker network ls + docker network inspect <name>查看创建出的虚拟网络信息，包括子网、网关等信息，以及对应的docker容器的IP地址等信息。

可以在docker容器内部运行cat /etc/resolv 可以看到nameserver指向了127.0.0.11，这实际上是docker-compose提供的DNS服务的代理，实际位置一般就是docker-compose创建的虚拟网络的网关地址。
