vim配置与hexo搭建

仅做个人记录

vim配置

此处直接使用的是wsl ubuntu22.04 自带的vim

配置按键映射

    inoremap jk <Esc>
    inoremap ' ''<ESC>i
    inoremap " ""<ESC>i
    inoremap () ()<ESC>
    inoremap [ []<ESC>
    inoremap { {}<ESC>i
    nnoremap o o<ESC>

安装vundle插件管理

将vundle插件clone到指定位置

    git clone https://github.com/VundleVim/Vundle.vim.git ~/.vim/bundle/Vundle.vim

在.vimrc文件中加入

    "这是vundle所需要的配置文件
    "--------------------------
    set nocompatible    
    filetype off

    " 设置Vundle管理vim插件 set the runtime path to include Vundle and initialize 
    set rtp+=~/.vim/bundle/Vundle.vim
    call vundle#begin()
    " 安装的vim插件写在下面

    
    " 安装的vim插件写在上面
    call vundle#end()     " required
    filetype plugin on    " required

安装插件，输入对应的地址或者名称

    Plugin 'Valloric/YouCompleteMe' "YCM"
    Plugin 'VundleVim/Vundle.vim'   " let Vundle manage Vundle, required
    Plugin 'scrooloose/nerdtree' "nerdtree"
    Plugin 'tpope/vim-surround' "自动环绕"
    Plugin 'jiangmiao/auto-pairs' "括号补全"
    Plugin 'https://gitee.com/linuor/gruvbox.git' "gruvbox主题"
    Plugin 'https://gitee.com/then_then/markdown-preview.nvim.git' "markdown预览"

进入vim，命令模式执行 

    ：PluginInstall

其中YCM耗时较长，且需要进一步配置（运行YCM文件目录下的 install.py文件（同样时间较长））

配置各个插件

    "这是markdwon预览的配置

    let g:mkdp_path_to_chrome = '/mnt/c/ProgramData/Microsoft/Windows/Start Menu/Programs/Microsoft Edge.lnk'

    let g:mkdp_auto_start = 0

    "

    " 自动关闭预览窗口

    let g:mkdp_refresh_slow = 0
    " 设置为 1 则只有在保存文件，或退出插入模式的时候更新预览，默认为 0，实时
    " 更新预览

    let g:mkdp_command_for_global = 0

    let g:mkdp_echo_preview_url = 1

hexo搭建

创建github库以username.github.io命名，设置好权限

安装nodejs和hexo和npm

    sudo npm install hexo-cli -g

使用

创建blog文件夹

    hexo init

生成

配置_config.yml文件

操作

    hexo clean
    hexo g

部署

    hexo d

本地查看 

    hexo s

可以将相关主题clone到themes文件加从而完成主题切换
