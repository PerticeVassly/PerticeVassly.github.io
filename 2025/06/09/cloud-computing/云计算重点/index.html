云计算复习

什么是云计算

云计算是一种基于互联网的计算方式，通过这种方式，共享的软硬件资源和信息可以按需求提供给计算机各种终端和其他设备

云计算是一种模型，可以实现随时随地，便捷的，按需的从可配置计算资源共享池总获取所需要的资源，资源可以快速的供给和释放，使得管理的工作量和服务提供者的接入降低到最少

计算模式的演进

集中计算 -> 效用计算 -> 客户机/服务器模式 -> 集群计算 -> 服务计算 -> 个人计算机与桌面计算-> 分布式计算 -> 网络计算 -> 软件即服务 -> 云计算

集中式计算：完全依赖一台大型的中心计算机的处理能力，即主机，与其相连的终端具有各不相同的非常低的计算能力

效用计算：考虑购买主机成本高，将IT资源包装成可以度量的服务提供给用户；

分布式计算：多个通过网络互联的计算机都具有一定的计算能力，他们相互之间传递数据，实现信息共享，协作共同完成一个处理任务

服务计算： 也就是软件即服务SaaS，将所有应用程序都作为服务提供，而不是购买软件。特点是不同服务相互独立，松散耦合，随意组合。重点是服务发现

云计算是一种效用计算 + 服务计算的方式（在传统的效用计算的基础上不仅考虑计费，还考虑it管理；不仅是软件还包括基础设施和平台）

一些分布式计算产品

计算机集群：将多个可行，静态的独立节点连接起来协作完成计算，主要是计算抽象。基本组件为计算节点，通信节点，网络接口卡，集群依靠中间件实现单系统镜像的支持，以单独计算机身份接入互联网

P2P网络：每一个节点即使客户端也是服务端，提供部分系统资源。主要是存储抽象。节点简单接入互联网，自由加入和退出，没有任何的中心协作或者中心数据库；没有专用的物理互联网络

计算网络：将多个异构计算机动态，紧耦合的整合在一起完成计算任务。接近“云”。提供了计算资源整合，作为一个集成的资源池。节点包括工作中站，服务器，集群和超级计算机。总体上是一个计算和数据网格P2P网络

云计算：最终的一种解决方案。包括大量计算节点，同构可控，专用内部网络，通过互联网对外服务，全面虚拟化，集中管理等等。

支持云计算发展有哪些助推力

网络宽带技术的提升：（传输效率提高）

技术的成熟度：（各种技术的融合，包括集成电路，计算芯片等）

移动互联网的发展：（物联网）

数据中心演变：（理念改变，技术成熟，需求增加）

经济因素：（降低成本节约资源）

大数据：（储存和计算大数据的要求）

持续优化的动力：节能降耗，降低维护成本，提高资产安全，提升信息系统的容灾备份能力

云计算的特征（其他计算范式的区别）

超大规模——物理特征

虚拟化——技术特征

按需服务——商业特征

通用，高可伸缩，极其廉价——得益于技术和模式

高可靠——目标特征

云的部署模型

共有云，社区云，私有云，混合云。

公有云:一般是第三方云提供者拥有可公共访问的云环境；

社区云:类似共有云，但是访问被限定为特定的云用户社区；

私有云:由一级组织单独拥有，利用云计算技术来访问组织内部的IT资源；

混合云:由两种或者更多不同云部署模型组成云环境

云的服务模型

IaaS 基础设施作为服务

PaaS 平台作为服务

SaaS 软件作为服务

由通用逐渐向专用转变。IaaS主要提供例如CPU/内存/硬盘; PaaS主要提供工具；SaaS主要提供软件

IaaS

指通过虚拟机的方式对外提供计算/存储/和网络能力

类似租用独立的计算机，需要用户自行解决多台机器之间的协同问题

优势：允许用户动态申请和释放资源，按使用量计费，具有更高的资源使用效率

PaaS

提供用户应用程序的开发和运行环境，包括应用编程接口和运行平台等等，支持应用充创建到运行整个生命周期需要的各种软硬件资源和工具（一般是提供经过封装的It能力，或者逻辑资源：数据库，文件系统，应用运行时环境）

其自身负责资源的动态扩展和容错管理，用户无需考虑节点之间的配合问题，但是用户自主权降低，需要按照给定的编程环境和模型构建应用

SaaS

通过标准的Web浏览器或者软件客户端访问云上应用

云计算三元认知论

一种商业模式，一种计算范式，一种实现方式

云计算和大数据/物联网/人工智能之间的关系

物联网采集数据，综合后的数据作为输入，由人工智能等等进行计算，由云计算提供算力，得到输出，这些原始数据和经过处理整合过的数据称之为大数据。根据计算得到的结果再动态的调配其中的过程

云计算中的风险和难题

难题

数据中心安全控制 -> 物理资源层

构建高度同构化的虚拟资源 -> 资源池层

高可用性的实现——资源管理/容错的实现——任务管理/节能——任务管理/云安全与隐私保护——安全管理 -> 管理中间件层

风险和挑战

增加了安全漏洞：用户将信任扩展到外部的云，不引入安全漏洞是非常困难的；恶意用户借此机会破坏和剽窃企业数据

降低了运营管理控制：云用户对于资源的控制显然低于云资源管理者，二者可能产生矛盾纠纷

不同的云提供者之间可移植性不同：…

多地区法规和法律问题：包括数据中心修建，潜在数据获得和公开

缺陷：

平台角度——没有统一的标准

计算角度——针对松耦合的数据处理

数据角度——管理和分析商业数据

资源继承角度——必须集中到云中

信息安全角度——暴露，丢失风险

云计算架构的二维视角

计算机演化以后最终的架构为 SOA

面向服务的体系结构是一个组件模型，它将应用程序的不同功能单元通过这些服务间定义良好的接口和契约建立起来联系。每个服务只做本职工作，将服务暴露出来给其他客户程序使用。

云计算采用的正是面向服务的计算架构 SOA

最易于理解的云计算二维视角：基础架构 + 应用程序；二者分别视为：后端（云组件） + 前端（给用户使用的服务）

最终呈现就好像一个Web服务一样

云应用程序的结构——云应用程序和传统操作系统上的应用程序结构有所不同

  • 传统操作系统上的应用程序：进程/线程/服务/shell，进程就是最终产品，每一个进程可以运行多个执行，一个线程对应一种服务

  • 云环境中的应用程序：应用程序本身，应用实例，服务，云命令行界面（控制端）

    • 应用程序只是云应用最终的鄂样子，实际上由多个运行的实例支撑。不同的实例运行在不同的服务器上，每一个应用实例提供一种或者多种服务，服务之间是相互独立的

逻辑云栈

一种并不严格的分类，每一层都提供一种抽象，最下面为物理硬件层

每往上一层，易用性就增加一分

以下为云栈的三层模式

基础设施即服务层，平台即服务层，软件即服务层

基础设施即服务层：不仅包括硬件设施，还包括虚拟化

平台即服务层：在虚拟化的IT资源上构建起应用程序的运行环境，对外提供的产品包括计算环境，云存储库

软件即服务层：基于网络进行远程控制访问商用软件/集中式管理而非分散在每个用户站点/单个实例多个租户架构——对多模型/按照用量计费

如何实现逻辑云栈

主要分成

SOA构建层 ： 封装成一个Web Services服务，并且纳入到SOA架构中

管理层 ： 主要是对云计算资源进行管理，包括资源的分配，调度，监控，计费等

资源池层 ： 将大量的相同类型的资源构成同构或者接近同构的资源池

物理资源 ： 具体的各不相同的物理资源

OpenStack

OpenStack是一个管理云计算中计算、存储和网络，甚至是应用的通用平台，其提供Web界面、命令行工具和应用程序接口（API）等使用接口。

一致性Hash算法

用在负载均衡等等管理场景中

问题：当服务器数量增加减少的时候，一个Key的原有Hash结果和新Hash函数结果不一样，我们希望之前的映射不变，但是后来的节点可以重新映射, 且分布依然均匀

解决方案：一致性Hash算法

将Cache和资源视为等同概念，都需要Hash。

  • 单调性：尽可能小改变已经存在的映射关系

  • 平衡性：尽可能均匀分布

具体操作：

  1. 将所有的节点映射到一个环上，环的大小为2^32

  2. 将所有的Key映射到同一个环上

  3. 节点的Hash值为环上的一个点，Key的Hash值为环上的一个点

  4. 根据key的值向后找一个最靠近的节点的Hash

对于增加，实际上就是增加节点与前一个节点之间的key的数据被分配到了增加的节点上

对于减少，实际上就是减少节点与前一个节点之间的key的数据被分配到了减少的节点下一个节点上

这样的引起的key的变动就比较小了，但是还是由分布不均匀的情况

解决方案是环上的所有节点都是虚拟节点，只要虚拟节点足够多，那对于key的变动就会非常小，同时分布也会非常均匀，这些虚拟节点再映射到真实的物理节点上去

优点：

只要给定Key就能查找对应的资源，无需指定IP等

能够很好的共享下层设施，形成一个统一的基于内容的命名层

网络自组织和自洽，降低了执行，维护，管理的代价

支撑云计算成为产品的具体的技术

分布式计算，分布式存储，虚拟化技术，容器技术，软件压缩技术，加密解密技术

Hadoop

主要应用与构建大型分部署集群（存储和计算），包括数据存储和数据挖掘两个部分

一些核心组件

  • 公共组件Common：其他模块的公共组件，定义程序员取得集群服务的编程接口，为其他模块提供API作用。提供包括公共API，本地Hadoop库，服务级别认证，HTTP认证

  • 分布式文件系统HDFS：一般是master/slave结构，文件分块存储：namenode/datanode

  • 分布式操作系统Yarn(资源管理系统)：资源管理（包括计算机资源，提供用户和程序访问资源的API）

Yarn

主要用于管理计算机资源，提供用户和程序访问系统资源的API

Client向集群提交作业后，ResourceManager负责集群资源的管理和调度，具体执行由ApplicationMaster负责，ApplicationMaster负责与ResourceManager和NodeManager通信，NodeManager负责具体的资源分配和任务执行，Container实际表示资源。ApplicationMaster是任务的执行和监管中心

    • 其核心问题是任务调度策略，ResourceManager的Scheduler模块支持插拔，包括两种策略容量调度算法（CapacityScheduler），公平调度算法（FairScheduler）

yarn的使用

yarn是可编程的，不仅仅支持自带的MapReduce，还可以定义算子，具有一套编程协议：Client负责提交任务， ApplicationMaster负责执行任务。同时ApplicationMaster还是NodeManager和ResourceManager之间的桥梁

虚拟化层级

硬件层级, 操作系统, 程序API, 应用程序

硬件虚拟化

优势

  • 封装性以虚拟机为粒度使得虚拟机运行环境保存很方便，保存很方便

  • 多实例：将多个物理服务器合并到少数服务器上以虚拟机运行

  • 隔离

  • 硬件无关性

  • 特权功能: 在这个层级添加特权功能，更加不容易被用户绕过

操作系统级虚拟化

基于Hype-visor的虚拟化技术，以及基于容器的虚拟化技术

不同层级的虚拟化之间的对比

包含ISA级别，运行时库，应用程序级虚拟化

OpenStack

是开源云计算平台，可控制整个数据中心的大型计算、存储和网络资源池。

管理员能够通过Web界面、命令行或API接口控制、配置资源；用户可以通过Web界面使用资源。

所有组件分布在四种节点上：云控制器，网络节点，计算节点，存储节点

云控制器上部署：数据库服务，身份验证，图片服务，网络服务，dashboard， 消息队列服务，编排引擎，计量服务，仪表盘

网络节点上部署：网络接口，连接和管理

计算节点上部署：计算服务，网络服务

存储节点上部署：对象存储服务（对象存储没有文件存储的层次结构，对象存储在扁平的命名空间——一种特殊的文件系统），块存储服务

四种节点如何通过物理网络连接在一起

  • 管理网络：OpenStack服务之间的通信，包括消息队列，数据库和API

  • 租户数据网络：为组合创建的虚拟网络提供物理路径

  • 存储网络：虚拟机和存储节点之间的物理连接

  • 对外的网络：互联网连接 + 对外的API网络

云计算国内外的差距

龙头企业： Amazon Microsoft Azure Google Cloud vs 阿里云 腾讯云 百度云

市场规模：市场趋于饱和创新型较强 vs 时长增速快集中度高

专注领域：国外主要是基础设施和平台服务，国内主要是应用服务（电商，金融，社交，游戏，视频）

应用场景：金融、电商、游戏、AI 模型训练、大数据分析更重视 API 式自动化部署 vs 政务、制造、教育、企业数字化转型为主，本地化部署+混合云更受青睐
