PV操作以及简单应用.md

基础知识

临界资源：

并发进程之间需要互斥使用的共享资源，称之为临界资源

临界调度原则：

  1. 一次至多一个进程可以进入临界区执行

  2. 如果有进程再临界区，其他试图进入的进程应等待

  3. 进入临界区的进程应该在有限时间内退出，以便让等待进程中的一个进入。

PV操作

P(s) 将s减1，若结果小于零，则调用p的进程被设置成等待信号量s的状态

V(s) 将s加1，若结果不大于0，则释放一个等待信号量s的进程，使其转换成就绪态。

上述操作应该在原语下执行

进程之间的唤醒

graph LR;
start ----> P1
start --block--> P2
start --block--> P3

P1 --> V1 
V1 --wakeup--> P2
P2 --> V2
V2 --wakeup--> P3
P3 --> V3

上述操作中的进入等待队列指的是进入等待态。

P操作对应运行态进入阻塞态（等待态）
V操作对应阻塞态进入就绪态

可以抽象为访问资源需要获取信号量.进程可以申请一个信号量，也可以释放一个信号量。信号量之间的传递（资源访问的调度）由进程之间的PV操作实现。

下面是写者优先的代码设计,其实就是把读者优先的设计加到了写者身上，不同在于写着不允许同时写但是读者可以允许同时读

独木桥问题

1

wait代表整个桥是否可以进入,用途是用来实现通车方向的转换，不然一直只有一个方向在通车。当count变为1的时候，wait被获取，另一方不可能获取wait从而全部被阻塞。直到这一方所有的车都走光了count重新归零(由最后一辆车完成)，wait才会被释放，此过程中由于等待一方count == 1这辆车一直被阻塞在wait处，所以其他的车也被阻塞在P(mutex2)上，当wait释放后，依次释放mutex2，在此过程中count变化直到1->0 此时wait在此被释放，通行方向再次转换。

P实际上是一个车的进程，利用ount的计数判断自己能不能走，利用wait判断需不需要通行转换。

3 三辆车为单位交错通行

可以看出需要对count进行处理

这里的意思是三辆车一组同时通过。

即使是最后一辆车，如果不是三车组，那么通行权也不交换。如果不是最后一辆车，但是是三车组起始的那一辆，那么由于判定某一侧仍然继续有来车，这一侧继续通信，直到这一侧没有来车了且这一辆走的也是三车组的最后一辆车了，此时交换通行权。
