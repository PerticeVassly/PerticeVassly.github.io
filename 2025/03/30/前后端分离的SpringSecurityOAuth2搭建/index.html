介绍

如何本地快速搭建一个极简的前后端分离的demo

前端: vue.js + axios

后端: Spring Security

由于是极简教程，此时直接使用已经写好的spring-boot-starter-oauth2-client的github认证功能

基本流程

OAuth2流程可以参考

https://www.ruanyifeng.com/blog/2014/05/oauth_2_0.html

前后端分离的OAuth2流程可以参考

https://blog.csdn.net/weixin_41866717/article/details/127092895

前端向后端发送第三方登录请求（以页面重定向的方式即window.location.href）重定向到后端对应处理的entry point即BACKEND_BASE_URL + "/oauth2/authorization/github", 后端处理请求并将页面重定向至github认证页面（即第三方授权页面），完成认证以后github服务器将浏览器页面重定向回redirect_url（即/login/oauth2/code/github 端点）并且附上code。后端处理对于这个端点的访问，使用这个code获取github的access_token，并且使用这个token调用github的api获取对应的用户信息，完成身份认证。

github

注册流程略

主要是 Home Page Url 和 redirect_url 两个配置

前者显示在github的第三方认证界面上，供用户查看你的应用到底是什么

后者主要用于检验redirect_url和发送OAuth2请求中的redirect_url是否一致（只有一致才返回code）

如果后端跑在8080上，可以写

HomePage Url ： http://localhost:8080

Authorization callback URL : http://localhost:8080/login/oauth2/code/github

Home.vue

前端点击登录的时候实际上是重定向页面到后端/oauth2/authorization/github 端点。这个端点又将页面重定向到github授权页面。github完成授权后，又将页面重定向到redirect_url上并且附上授权码，后端再利用授权码获取token进行身份认证。

spring项目框架略

application.yml

client-id: ${GITHUB_CLIENT_ID} 和 client-secret: ${GITHUB_CLIENT_SECRET} 创建github OAuth2应用的时候会给出

SecurityConfig.java

UserController.java

Usercontroller： 在认证完毕以后，获取认证信息中的name并且返回

SecurityConfig： 主要是安全配置。

后端认证端点（前端直接重定向的） /oauth2/authorization/github 就是将clientid redirect_url(回调端点)等信息组合起来发给github授权

后端的回调端点 /login/oauth2/code/github 的对应处理逻辑其实就是把clientid clientsecret 以及返回的来的code组合起来发给github获取token，再调用api获取对应的用户信息

但是上述内容已经由spring-security-oauth2-client写好并默认暴露在这两个端点了。

前后端分离的一些注意点

spring-security-oauth2-client 默认在获取user信息，登录成功以后返回后端主页(localhost:8080) 所以要修改，使其重定向到前端地址

首先这里只是一个简单demo，所以没有使用JWT这样的形式，而是直接用的Jsession，（不安全）所以需要使用

告诉浏览器加入Jsession作为header

Spring Security 正是在上述登录完成之后将这个JsessionID发给前端浏览器的. 浏览器后续可以凭借此字段通过身份认证

此外，还需配置跨域请求，参考上述后端代码。
