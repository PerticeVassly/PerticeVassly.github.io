SQL

SQL(Structured Query Language) 是关系数据库的标准语言

SQL是一个通用的功能极强的关系库数据语言

SQL与关系数据库的三级模式

基本表:

  • 本身独立存在的表

  • 一个关系对应一个基本表

  • 一个或者多个基本表对应一个纯存文件

  • 一个表有若干索引

储存文件:

  • 逻辑结构组成了关系数据库的内模式

  • 物理结构对于用户是隐藏的

视图

  • 从一个或者是几个基本表导出的表

  • 数据库中只存放视图的定义而不存放视图对应的数据

  • 视图是一个虚表

  • 用户可以在视图上定义视图

数据查询

SELECT语句的含义是,跟根据WHERE子语句的条件表达式从FROM子句指定的基本表,视图,派生表中找到满足条件的元组,再按照SELECT子句中的目标表达式选出元组中的属性值形成结果表

如有有GROUP BY 子句,则按照所给的列的值进行分组,通常在每一组中使用聚集函数.如果有HAVING短语,则只有满足条件的组才输出

ORDER BY表示结果表要按照所给的列的值升序或者降序排序.

连接谓词中的列的名称称之为连接字段,各个连接字段的类型必须是可比的,名字不需要相同

嵌套查询

在sql中一个SELECT-FROM-WHERE语句称之为一个查询块,将一个查询块嵌套在另外一个查询快的WHERE子句或者HAVING短语的条件中嵌套查询

上层称之为父嵌套,下层称之为子嵌套

子嵌套不能用ORDER BY子句 ORDER BY子句只能对最终查询结果排序

子查询的条件依赖于父查询这类子查询称之为相关子查询否则称之为不相关子查询

ANY 和 ALL

子查询返回单值的时候可以使用比较运算符，但是返回多值的时候要使用ANY或者ALL修饰符。而是用ANY ALL谓词的时候必须要使用比较运算符。

使用ANY和ALL谓词的时候必须同时使用比较运算符

EXISTS

带有exists的子查询不返回任何数据,只产生逻辑真值true和false

集合查询

集合的操作

UNION INTERSECT EXCEPT

UNION 会自动将查询加过合并起来,系统自动去掉重复的元组

UNION ALL 将多个查询结果合并起来保留重复元组

基于派生表的查询

子查询不仅可以出现在WHERE子句中，还可以出现在FROM子句中，此时子查询生成的临时派生表成为主查询的查询对象。

通过from子句子生成派生表的时候，AS关键字可以省略，但是必须为派生关系指定一个别名

数据更新

插入

对于INTO子句中没有出现的属性列，新元素在这些列上面会取空值。但是要注意NOT NULL的属性列不能取空值

如果INTO子句中没有属性列，说明是是对整个表进行操作，此时VALUES要和表中各个项一一对应。

插入子查询结果

子查询可以嵌套在INSERT语句中以生成要插入的批量数据（不要VALUES）直接换成子查询

修改

子查询也可以嵌套在UPDATE语句中，用以生成构造修改的条件

删除

DELETE语句本质上是删除元组。如果省略WHERE语句删除表中的所有元素，那么表的定义还在数据字典中，因为只是删除元组

子查询也可以嵌套在DELETE语句中用于构造执行删除操作的条件

空值的处理

判断空值

使用IS NULL IS NOT NULL

空值的约束条件

属性定义中有NOT NULL约束条件的不能取空值，加了UNIQUE限制的属性不能取空值，码属性不能去空值

空值的运算

空值与另外一个值的算术运算的结果为空值，空值与另外一个值的比较运算结果为UNKNOWN

由此扩展出三值逻辑运算表

视图

视图是从一个或者几个基本表中导出的表，其内部不存放数据，只存放视图的定义，这些数据依然存放在原来的基本表中

定义

WITH CHECK OPTION表示对于视图进行UPDATE，INSERT和DELETE操作的时候要保证更新插入或者删除的行满足视图定义中的谓词条件。

组成视图的属性列名要门全部省略要门全部指定，没有第三种选择。如果省略了视图中的各个属性列名，则由隐含该视图由子查询中SELECT子句目标列中的各个字段组成。但是以下情况需要指定组成的视图的全部列名

（1）某个目标列不是单纯属性名，而是聚集函数或者列表达式

（2）多表连接的时候选出了几个同名列作为视图的字段

（3）需要在视图中为某一个列启用新的更加合适的名字

如果一个视图是从单个表导出的，并且只是去掉了基本表的某些行和某些列，但是保留了主码，则称这类视图为行列子集视图

还可以使用带有聚集函数和GROUP BY子句的查询来定义视图，这种视图称之为分组视图

查询视图

和查询基本表一样

实际上数据库先取出视图的定义，然后把定义中的子查询和用户查询结合起来转为为等价的对基本表的查询，然后再执行

称之为视图消解(view resolution)

一般来说当前多数关系数据库对于行列子集视图查询均能进行正确转换，但是对于非行列子集视图的转换不一定能做转换
