可能的拷打问题

你提到使用了 SpringCloud 实现配置分发和微服务管理，具体用了哪些组件（比如 Nacos、Eureka、Gateway、Config 等）？请详细描述它们的作用和使用场景。

使用了 Spring Cloud Config 作为集中式配置管理服务。各微服务模块（如社区模块、图片展示模块）在启动时会向 Config Server 拉取相应的配置。没有做具体生产环境和开发环境的区分。没有使用git这样的配置存储

所有微服务在启动时通过注解@EnableDiscovery自动注册到 Eureka Server上。

网关层主要是使用了Netflix的Zuul网关。主要提供前端资源的访问和，转发功能。网关首先从config中拿去各个要转发的路由信息，然后进行转发，用了一个Command注解（不过这个似乎是自动返回的），所有的（ribbon（负载均衡）方式转发的服务统一设置熔断超时20s，和zuul本身也要设置20s超时限制））

鉴权是如何设计的？

鉴权方面主要是使用OAuth2 + jwt这样的鉴权方式。可以自己创建登录用户，也可以关联类似于github这样的第三方登录。OAuth2鉴权验证以后返回JWT（id 邮箱，角色）（如果说数据库中有则直接返回），JWT本地存储到数据库MYSQL中。并且设置过期时间（Springboot的schedule）网关仅仅只是转发。当时写了一个基于JWT识别用户身份的限流，方面针对高访问量api，比如下载，上传，新增评论等，使用Google的Guava的RateLimiter进行限流。

是要使用的RBAC（RoleBased-Accesscontrol）权限控制，其实主要就是访客，普通用户，和管理员。对应的数据库表主要是用户表，角色表，权限表，（用户角色关联表，角色权限关联表）

JWT没有设计续签，如果过期了就需要重新登录。（非要设计可以设计一个队列存储目前或缺的所有的jwt）

Jacoco 与 Mockito 测试

单元测试主要是对一些业务逻辑进行测试，用mockmvc这样的工作做一些集成测试。

何时mock何时真实调用，数据库相关的或者外部服务调用的都使用mock。如果是一些业务逻辑的测试就使用真实的调用。

Jacoco，没有做详细的指标，仅仅作为参考。而且jacoco的版本问题，还导致环境配置有bug，最后只是生成一个报告

Redis 缓存设计

主要是图片展首页图片的url（因为是用obs存储的），以及社区的设点帖子内容

不一致问题，主要是更新完数据库再更新缓存。

MySQL 事务的四大特性（ACID）是如何实现的？分别依赖哪些机制？

A：原子性 各类回滚操作

C：一致性 外键约束，触发器，锁机制等等

I：隔离性 并发控制，锁机制

D：持久性

向量检索流程是如何工作的？请从用户提问到结果返回的整个 pipeline 简要说明。

是针对爬取的数据先做切分，然后调用大模型进行embedding，返回值存储到向量数据库中，再进行检索。使用匹配的形式

多mod依赖关系。主要是neo4j,具体类型MATCH (start:Mod {name: 'ModA'})-[:DEPENDS_ON*1..]->(dependency:Mod)RETURN start.name AS StartMod, dependency.name AS Dependency
