原型模式

原型模式(Prototype Pattern)

原型模式是一种对象创建型模式，用 原型示例指定创建对象的种类，通过复制这些原型创建新的对象

Prototype Pattern: Specify the kind of objects to create using a prototypical instance, and create new objects by copying this prototype.

模式结构

classDiagram
    
    class Prototype{
        Clone() : Prototype
    }
    class ConcretePrototypeA{
        Clone() : Prototype
    }
    class ConcretePrototypeB{
        Clone() : Prototype
    }
    class Client{
        prototype : Prototype
        copy : Prototype
        operation()
    }

    Prototype <|-- ConcretePrototypeA
    Prototype <|-- ConcretePrototypeB
    Client --> Prototype

主要包括

Prototype：抽象原型类

ConcretePrototype：具体原型类

Client：客户类

Java中java.lang.Object类提供了一个clone()方法，可以将一个Java对象复制一份

实现克隆的java必须实现一个Cloneable接口（marker） 如果一个类没有实现这个接口，调用clone()方法会抛出CloneNotSupportedException异常

深克隆和浅克隆

一般来说clone方法满足

x.clone() != x

x.clone().getClass() == x.getClass()

x.clone().equals(x)

优缺点

优点：

简化对象的创建过程

动态增加/将减少产品类

简化的创建结构

保持对象的状态

缺点：

  • 需要为每一个类配置一个克隆方法，且需要对类的功能进行通盘考虑，对已有的类进行改造的时候，违反了开闭原则

  • 深克隆比较复杂

适用场景

创建新对象成本比较大

系统要保存对象状态（享元）

避免使用分层次的工厂类来创建分层次的对象

模式扩展

带有原型管理器的原型模式

相似对象的复制
