01背包

给定一组物品，每种物品都有自己重量和价值，在限定得重量内，如何选择才能使得物品得总价值最高

定义dp[i][j]为前i个物品在重量不超过j的情况下的最大价值（注意这里说的“装入”并不是说明真正的就装入了，只是说明如果要装）

dp[i][j] = max(dp[i-1][j], dp[i-1][j-w[i]] + v[i])

这是一个二维dp，但是仔细观察发现dp[i][j]只和dp[i-1][j]和dp[i-1][j-w[i]]有关，也就是说i这一个维度只需要前一个维度的值即可，因此可以将其压缩为一维

同时由于j和j-w[i]，因此必须逆向遍历

复杂度为O(nW)，n为物品个数，W为背包容量

完全背包

区别在于每种物品可以有无限多个

依然是dp[i][j]只不过这里的i指的是第i种物品。

dp[i][j] = max(dp[i-1][j], dp[i][j-w[i]] + v[i])

此时减去-w[i]的部分仍然是i，因为可以选择当前物品的多个

依然存在上述维度压缩的问题，因此可以将其压缩为一维

此时必须正向枚举，因为dp[i][j]的值依赖于dp[i][j-w[i]]，后者必须先行计算过才能使用

或者可以引入每种物品的数量限制, k = j / w[i]

dp[i][j] = max{(dp[i-1][j − k*w[i]] + k*v[i]) for every k}

其实思路就是，既然上述背包问题种涉及到dp[i]了，那么我就穷举涉及到i的所有情况，这样的我递归式子种就没有i的项了

或可以从每种物品的最大数量入手，i种物品可以转换为W/w[i] 种费用和价值均不变的物品.

但是这样仍然有优化空间，毕竟完全相等的拆分最后会导致N变得庞大，可以使用进制的方法奖把第 i 种物品拆成重量为 w_i2^k、价值为 v_i2^k 的若干件物品

多重背包

每种物品有有限个，一共有N种物品

这个时候就上上述完全背包的后两种思路

引入N种物品具体装入多少件

转为01背包

完全展开或者按照进制展开（但是此时进制展还要考虑允许的最大数量）

5. Longest Palindromic Substring

参考：

https://leetcode.com/problems/longest-palindromic-substring/solutions/4212564/beats-96-49-5-different-approaches-brute-force-eac-dp-ma-recursion/?envType=study-plan-v2&envId=top-100-liked

Given a string s, return the longest palindromic substring in s.

最朴素的想法是遍历所有的subString，判断是否是回文串，复杂度O(n^3)

但是实际上遍历过程重复检验了一些字串很多次。因此考虑是否能通过已有的字符串推断出其他的字符串是否是回文串。

首先基础条件是一个单独的char是回文串，两个相同的char也是回文串。由于一个回文串一定有一个中心（空或者是一个char），那么可以依据这个中心推断出所有的回文串。复杂度O(n^2)

实际上是遍历了所有可能的中心并且向外expand了所有可能的字符串。Beats 58%

或者使用dp方法

O(n^2)的复杂度，Beats 34%

或者

Manacher算法，提供了O(n)的复杂度，具体参考manacher算法一文,(但是最后竟然只beat了 25%)

64. Minimum Path Sum

Given a m x n grid filled with non-negative numbers, find a path from top left to bottom right, which minimizes the sum of all numbers along its path.

Note: You can only move either down or right at any point in time.

思路和上面的unique path类似，这里使用贪心的方法，每一步都选择最小的路径

证明：很显然如果最小路径中又一个节点不满足其也为起点到终点的最小路径，那么就可以将其替换为最小路径中的节点，形成一个更小的路径

ps: 初始化和循环层数的问题需要注意一下
